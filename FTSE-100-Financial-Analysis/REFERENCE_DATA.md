# Reference data (UK market terminal realism)

This repo ships small **reference** datasets under `data/reference/` to make the project feel like a
real UK-market analytics terminal.

These datasets are intentionally:
- **Offline-first** (builds run without internet access)
- **Portfolio-safe** (small, readable, and easy to explain in interviews)
- **Replaceable** (you can refresh them later with real feeds)

> Not financial advice.

---

## 1) FTSE 100 constituents universe

**File:** `data/reference/ftse100_constituents_universe_snapshot.csv`

Columns:
- `company_name` — display label
- `epic` — LSE EPIC ticker (as commonly shown on public sources)
- `ticker` — Yahoo-style ticker (best-effort conversion), e.g. `AZN.L`, `BT.A.L`
- `icb_sector` — sector label as shown on the source table
- `sector` — mapped to a compact 10-sector palette for dashboards
- `index_weight` — **heuristic / demo weights** (sum to 1.0)
- `weight_rank` — rank by `index_weight`
- `as_of_date` + `source` — snapshot metadata

Why we ship weights:
- to enable **index decomposition** and **sector contribution** dashboards
- to power “Top movers” and “Heatmap” views with realistic-looking sizes

### Refresh strategy (optional)

The repo includes a best-effort Wikipedia loader:

```bash
python scripts/v2_build_all.py --constituents-source wikipedia --weights-method equal
```

For production weights you’d typically use:
- FTSE Russell licensed files (preferred / official)
- a symbology service + market-cap computations (approximate)

## 2) Macro calendar (stub)

**File:** `data/reference/uk_macro_calendar_stub.csv`

A small curated calendar used for:
- dashboard overlays (volatility around events)
- “what moved the market?” storytelling

This is a **stub** calendar: it is designed for realism, not exact scheduling.

---

## 3) Earnings calendar (stub)

**File:** `data/reference/ftse100_earnings_calendar_stub_top25.csv`

We intentionally ship only the **top-25** names for dashboard readability.
In production, you’d pull full earnings calendars from a vendor and join by ticker.

---

## 4) Market news headlines (stub)

**File:** `data/reference/market_news_headlines_stub.csv`

This dataset is included for “terminal feel”. It can be merged into the unified
`events_calendar` if you run:

```bash
python scripts/v2_build_all.py --include-news
```
